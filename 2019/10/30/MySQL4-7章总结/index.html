第五章模糊查询与分组查询

1.模糊查询

1.使用like关键词进行模糊查询；

表达式为：where 字段 LIKE 通配符字符串；

2.常用的通配符还有”%”、”_”;

%用于通配多个字符；表示了0~n个任意字符。

例如: 王%代表以”王”开头的任意字符串;

​ “%了”代表了以”了”结尾的字符串;

查询studentinfo表中所以姓李的所有学生信息。

_用于通配单个字符，表示一个字符；

若需要表示两个字符的匹配，则需要使用两个”_”通配符；

例如：

“刘_”代表了以刘开头的单个字符串；

sum()函数用于对数据求总和；

min()函数用于求最小值；

count()函数用于求个数；

count(字段)：计算查询结果中指定字段的总行数，在字段为null时，不会统计个数；

count(*)：计算数据表或查询结果集的总行数，不管某一列或多列是否为空值；

例如：

如果查询结果有20条，但A字段有3个空值，那么count(A)的结果就是17；count(*)的结果就是20;

第六章多表连接查询

1.连接查询

常用的连接查询有：内连接，外连接，交叉连接；

1.交叉连接是将多张表中的数据一一对应连接在一起，形成结果集的数据相当于多张表的数据行数“相乘”之后的结果。

交叉表中没有where字句；

例如：将T_A表名改成A表，把T_B表名改成B表，并查询两个表中的所有信息；

实际上，select语句中可以实现两张表以上的表的连接查询，其创建规则与两个表连接相同，只需要在where条件中增加一个AND复合条件；

例如：三张表，学生姓名存放于studentinfo表中，科目名称存放于subject表中，考试成绩存放于exam表中

第七章SQL高级子查询

7.1简单子查询

一．什么是子查询？

所谓子查询，是指包含在一个INSERT、DELETE、UPDATE、SELECT命令中的SELECT查询。

也可以理解为增删改查命令中嵌套着查询命令。

二．子查询在WHERE语句中的一般用法：

SELECT … FROM 表1 WHERE 字段1 >(子查询)，外面的查询称为父查询，括号中嵌入的查询称为子查询 ，UPDATE、INSERT、DELETE一起使用，语法类似于SELECT语句

将子查询和比较运算符联合使用，必须保证子查询返回的值不能多于一个；

7.2 高级子查询

一．EXISTS子查询

如果子查询的结果非空，即记录条数1条以上，则EXISTS （子查询）将返回真（true），否则返回假(false) ，EXISTS也可以作为WHERE 语句的子查询，但一般都能用IN子查询替换

通过在子查询中使用EXISTS子句，可以对子查询中的行是否存在进行检查

二.使用IN和NOT IN的子查询

1.IN子查询

IN子查询后面可跟随返回多条记录的子查询，用于检测某列的值是否在某个范围；

–使用IN后面的子查询可以返回多条记录。常用IN替换等于(=)的比较子查询

IN：在指定项内，同 IN(项1,项2,…)。

Select StudentName from Students where StudentId IN
(Select StudentId from ScoreList where SQLServerDB>80)

2.NOT IN 子查询

–查询没有参加考试的学员、姓名？

Select StudentName from Students where studentId not in
(Select StudentId from ScoreList where SQLServerDB>80)

注意：

‘IN’ 是 ‘= ANY’ 的简写，’NOT IN’ 是 ‘<> ALL’ 的简写;

三.使用ALL,ANY/SOME的子查询

在使用子查询语句的过程中经常使用ANY、SOME、ALL（另外还有一个IN）关键字作为筛选条件。

ANY，SOME，ALL关键字的使用

1.ANY：与比较操作符联合使用，表示与子查询返回的任何值比较为 TRUE ，则返回 TRUE 。

2.ALL：与比较操作符联合使用，表示与子查询返回的所有值比较都为 TRUE ，则返回 TRUE 。

特殊情况:

如果 table2 为空表，则 ALL 后的结果为 TRUE；

3.SOME：ANY 的别名，较少使用。
